def MutableList __join__ (MutableList this, List other) (bound);

This method is usually called as the ~ operator.

The join operator is used to create a new list that contains all the elements of both the original lists.

The join operator can be used like this:

combinedCollection := firstCollection ~ secondCollection;

Note that the join operator joins two collections. If all you want to do is add one element to an existing collection, use the push method.

Assign new values to a range of items

This method can operate in three distinct modes:

The first mode, simple mode, is when the slice to replace and the supplied replacement have the same number of items. In this case, the slice operation will simply replace the old values with the new ones in place and the list will be otherwise unchanged. Some examples of simple mode usage:

// Simple replacement
[1,2,3,4,5][1..2] = [22, 33]; // Result: [1, 22, 33, 4, 5]
// In simple mode, it is allowed to use any step in the range. This
// will replace the even items
[1,2,3,4,5][1|2..4] = [22, 44]; // Result: [1, 22, 3, 44, 5]
// We can use open ranges, and the end of the range will simply be
// the list boundary. This will replace the even items.
[1,2,3,4,5][1|2...] = [22, 44]; // Result: [1, 22, 3, 44, 5]

The second mode, complex mode, is when the slice to replace and the supplied replacement have different lengths. In complex mode, the step of the slice range must always be ±1. In this case, the slice operation will remove the old range and replace them with new values, which result in a list with a different length.

// Complex replacement, change the list length
[1,2,3,4,5][1..2] = [11, 22, 33, 44]; // Result: [1, 11, 22, 33, 44, 4, 5]
// In complex mode, it is not allowed to use any step in the range.
[1,2,3,4,5][1|2..4] = [11, 22, 33, 44]; // This will result in an error
// We can use open ranges, and the end of the range will simply be
// the list boundary. This will replace the last element with the new list.
[1,2,3,4,5][-1...] = [11,22,33,44]; // Result: [1, 2, 3, 4, 11, 22, 33, 44]

The final mode, erase mode, is when the supplied replacement has a length of zero. In this case, all those elements will be removed from the list, and the list will thus be shortened.

// Erase mode, change the list length
[1,2,3,4,5][1..2] = «Int»[]; // Result: [1, 4, 5]
// Erase mode, we can use any step in erase mode
[1,2,3,4,5][1|2..5] = «Int»[]; // Result: [1, 3, 5]
// Erase mode, open ranges work as exected
[1,2,3,4,5][1|2...] = «Int»[]; // Result: [1, 3, 5]

def MutableList sort (List this, def Int (Object first, Object second) fun = ?) (bound);

Sort the list. If the original list is mutable, it is sorted inplace, otherwise a new list is returned.

The current implementation is just a placeholder. It is a standard Quicksort with a non-random pivot, meaning it does not provide a stable sort, and it is easy to trigger O(N^2) worst case behavior by choosing the right input. Also, the current implementation does not support templating, so the returned, sorted list is simply a list of objects.
